Expand description

Shared view over VE operand types that carry a branch-gating predicate plus the rhs value(s) used at that branch. Implemented by BinaryOperandTag (one rhs) and TernaryOperandTag (two rhs) so VE apply helpers can iterate either kind through one code path.

Mainstream is not part of this trait. A ternary op takes three positional inputs (mainstream, operand0, operand1), but the trait only exposes the two rhs values that vary per branch — mainstream is a tensor-level input passed separately to the apply helper.

Operand1 is the type of the secondary rhs: () for BinaryOperandTag and f32 for TernaryOperandTag, so callers that know the concrete type get a typed value without Option/expect.
