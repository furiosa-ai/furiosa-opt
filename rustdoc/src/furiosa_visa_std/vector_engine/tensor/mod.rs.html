1//! Vector Engine tensor types for pipeline operations.
2//!
3//! This module provides tensor types that track VE pipeline stages at compile time
4//! using the typestate pattern. Operations are expressed as method chaining, with
5//! stage transitions enforced by the type system.
6//!
7//! # Key Types
8//! - `VectorTensor`: Unified tensor type for all VE pipeline stages
9//! - `VectorInitTensor`: VE input after `vector_init()`, before choosing the first block
10//! - `VectorTensorPair`: Pair of tensors for two-group (interleaved) operations
11//! - `VectorInterSliceReduceTensor`: After inter-slice reduce
12//!
13//! # Initialization / Entry (on CollectTensor / AccumulationTensor)
14//! ```text
15//! vector_init()                              → VectorInitTensor
16//!   ├─ vector_intra_slice_branch(BranchMode) → VectorBranchTensor           { VeOrder::IntraFirst }
17//!   ├─ vector_intra_slice_unzip(...)         → VectorTensorPair             { VeOrder::IntraFirst }
18//!   └─ vector_inter_slice_reduce(op)         → VectorInterSliceReduceTensor { VeOrder::InterFirst }
19//! ```
20//!
21//! # Exit
22//! `vector_final()` exits the VE pipeline and returns a `VectorFinalTensor`.
23//! From there, `commit`, `cast`, `to_vrf`, or `transpose` are available.
24//!
25//! # Four Pipeline Paths
26//! ```text
27//! Path 1 (intra only):     vector_init → vector_intra_slice_branch → [intra stages] → vector_final
28//! Path 2 (intra → inter):  vector_init → vector_intra_slice_branch → [intra stages] → vector_inter_slice_reduce → vector_final
29//! Path 3 (inter only):     vector_init → vector_inter_slice_reduce → vector_final
30//! Path 4 (inter → intra):  vector_init → vector_inter_slice_reduce → vector_intra_slice_branch → [intra stages] → vector_final
31//! ```
32//!
33//! # Intra-Slice Stage Flow (IntraSliceStage)
34//! ```text
35//! Branch → Logic → Fxp → FxpToFp → Narrow(Way4) → Fp → IntraSliceReduce → FpDiv → Widen(Way8) → FpToFxp → Clip
36//! ```
37//!
38//! # Filter
39//! `vector_filter` is available after intra-slice stages and after inter-slice reduce
40//! (when `VeOrder::IntraFirst`). It applies VE-level branch filtering.
41//!
42//! # VeOrder Tracking
43//! The `const VE_ORDER: VeOrder` parameter tracks which unit was entered first:
44//! - `IntraFirst` — after `vector_intra_slice_branch` / `vector_intra_slice_unzip`
45//! - `InterFirst` — after `vector_inter_slice_reduce` from `VectorInitTensor`
46//!
47//! Once set, `VeOrder` is preserved through all subsequent operations.
48//! It prevents re-entering the same VE block as an initial entry path.
